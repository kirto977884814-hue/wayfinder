# Session 1：人生时间线

> 本文件是访谈者的「地图」，不是照念的剧本。访谈哲学、硬规则、挖掘技巧见 `../SKILL.md`。

## 目标
建立用户的成长轨迹。

## 开场白（破冰）
> "我们这会儿不聊正事，就当朋友瞎聊。我想先大致了解下你是怎么一步步长到现在的——不用很有条理，想到哪说到哪。"

从轻的切入，建立安全感。迷茫的用户尤其需要先放松下来才聊得动。

## 核心问题（地图，按需自然带入，绝不连续抛出）
- 小时候是什么样的人？
- 小学最喜欢什么？
- 初中最大的变化是什么？
- 高中最难忘的经历是什么？
- 大学是如何度过的？
- 目前处于什么状态？

单轮只问一个；根据回答追问**原因、情绪、转折点**。

## 覆盖检查清单（收尾前确认都碰过）
- [ ] 童年
- [ ] 小学
- [ ] 初中
- [ ] 高中
- [ ] 大学 / 近年
- [ ] 当前状态

没碰到的，温和地带回来（"对了，初中那会儿你是怎样的？"）。

## 收尾合成
Session 结束前做一次合成：
1. **概览**（叙事）："听下来，你的人生主线大概是……"
2. **结构化信号**：阶段关键词（如"童年内向 / 初中转折 / 大学迷茫"）
3. **确认**："我这样理解对吗？有要改的吗？"
4. 确认后，套 `../templates/LIFE_TIMELINE.md` 写入：结论层（阶段关键词）+ 证据层（每个时期的事件，带来源）。

## 输出
`LIFE_TIMELINE.md`
